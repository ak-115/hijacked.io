    Using Vega To Scan Your Website

    What is Vega??

    Vega Vulnerability Scanner, a graphical web-auditing tool developed by security company Subgraph, is a free and open-source tool that can be helpful in various ways. It offers features like proxy scanning, but in this discussion, we will specifically highlight its automated security testing capability, which aids in identifying and validating vulnerabilities such as SQL injection, cross-site scripting (XSS), inadvertent disclosure of sensitive information, and many others.

    Getting started with Vega

    If you're using a newer version of Kali Linux, the Vega Vulnerability Scanner may not come preinstalled like it did on older versions. Therefore, you may need to install it separately to access its functionalities."

    In earlier iterations of Kali, which came with Vega preloaded, users could access Vega by navigating to Applications -> Web Application Analysis -> Vega. However, in the current version, you would need to manually change directory (cd) into the Vega folder and then use the ls command to view its contents.

    To initiate Vega, the required tool inside the designated folder, you can launch it using the "Vega" command.

    If the process was successful, you should be able to view Vega's initial screen as it loads up. If it did not work, please verify that you are using the correct version of Java.

    Configure Vega

    Once we launch an application for the first time, we prefer to check out the available preferences and options. To access them, I click on the "Window" menu located in the top-left corner and then select "Preferences."

    How to scan a website using Vega??

    With Vega successfully installed and configured, we are all set to begin scanning a website. To initiate the scanning process, you can either use the Control-N shortcut, click on the Start New Scan icon located in the top left corner, or select "Scan" from the menu bar and then choose "Start New Scan." Vega will then prompt you with the Select a Scan Target window, where you can enter your target URL in the designated box under "Scan Target," and click "Next.

    Vega offers a variety of modules that are specifically designed to identify common web server vulnerabilities, including SQL injection, cross-site scripting (XSS), and XML injection vulnerabilities. To enable all of the scanner modules, you can ensure that "Injection Modules" is checked. Alternatively, if you only want to enable specific modules, you can expand the "Injection Modules" option and select the ones you want, while deselecting the ones you don't need.

    You could click "Next" a few times to adjust Authentication Options or Parameters, but we don't need to. So instead, click "Finish" to start scanning the website.

    A scan can take anywhere from two minutes to eight hours to complete depending on the website's size.

    How to interpret Vega alerts??

    After completing the scan, Vega will provide a summary of the alerts in a clear and concise manner. If Vega generates a "High" alert, there's no need to panic as Vega's modules can be sensitive and may occasionally produce false positive alerts for vulnerabilities that may not actually exist. False positives are not necessarily negative. However, it is recommended to thoroughly review the report and manually investigate each alert.

    Nmap Command in Linux with Example

    Nmap is Linux command-line tool for network exploration and security auditing. This tool is generally used by hackers and cybersecurity enthusiasts and even by network and system administrators. It is used for the following purposes:

    • Real time information of a network
    • Detailed information of all the IPs activated on your network
    • Number of ports open in a network
    • Provide the list of live hosts
    • Port, OS and Host scanning

    Installing Nmap Command

    In case of Debian/Ubuntu

    Sudo apt-get install nmap

    In case of CentOS/RedHat

    Yum install nmap

    Working With Nmap Command

    1. To scan a system using its hostname or IP address, use the command "nmap" followed by the hostname or IP address.

    Now, scan using IP address.

    2. To get more detailed information about a remote computer, use the "-v" option.

    3. To scan multiple computers at once, type their IP addresses or hostnames after "nmap".

    4. To scan a whole range of IP addresses, use the "*" symbol.

    5. To scan a specific range of IP addresses, type the first and last IP addresses separated by a hyphen.

    To detect firewall settings, use the "-sA" option.

    To find hostnames for a given IP address, use the "-sL" option
